在某些情况下，你可能会遇到 -sh: 178: source: not found 错误，这通常意味着你正在使用的 shell 不支持 source 命令。source 是一个 bash 内建命令，但在某些 POSIX 兼容的 shell（如 sh 或 dash）中，它被替换为 .（点命令）。

要解决这个问题，可以尝试以下几种方法：

1. 使用 . 命令代替 source
如果你的 shell 不支持 source 命令，可以使用 . 命令，它具有相同的功能。
```
. ~/.bashrc
```
2. 确保使用的是 bash shell
如果你需要使用 bash 特有的功能，可以切换到 bash shell。你可以通过以下命令启动一个新的 bash 会话：
```
bash
source ~/.bashrc
```
3. 修改默认 shell
如果你希望永久使用 bash 作为默认 shell，可以使用 chsh 命令更改默认 shell：
```
chsh -s /bin/bash
```
之后，重新登录会话，以使更改生效。

4. 检查脚本的 Shebang 行
如果你在运行一个脚本，确保脚本的 Shebang 行（位于文件的第一行）正确指向 bash。例如：
```
#!/bin/bash
```

# bashrc配置

## .bash_profile 内容
~~~
if [ -f /home/hanfy/.bashrc ]; then
    . /home/hanfy/.bashrc
fi
~~~

    . /etc/bash_completion

## sudo systemctl stop yunshu-da命令无效
sudo systemctl stop yunshu-da 无效，说明进程 yunshu-da 可能不是通过 systemd 管理的服务启动的，而是通过其他方式（比如登录后自动运行、图形界面启动，或以守护进程后台运行）启动的。
输入下面命令：
```
systemctl --user list-units | grep yunshu
```
>输出：
 yunshu-screenshot.service    loaded active     running   [Linux YunShu Screenshot Server Service] 

说明 yunshu-da 是通过当前用户的 systemd 用户服务（User Service） 启动的，而不是系统级的服务。


## 杀掉云枢
1. 停止当前用户的该服务：
systemctl --user stop yunshu-screenshot.service
2. 禁用开机启动（取消自启）：
systemctl --user disable yunshu-screenshot.service
3. 验证服务是否已停止：
   systemctl --user status yunshu-screenshot.service
   你应该看到状态变为 inactive (dead)。
4. 查看这个服务是从哪来的？
	systemctl --user cat yunshu-screenshot.service
5. . 彻底删除这个服务,你需要 root 权限删除系统提供的 .service 文件：
sudo rm /etc/xdg/systemd/user/yunshu-screenshot.service
6. 重启或执行以下命令刷新用户服务状态：
systemctl --user daemon-reexec
7. 彻底干掉 yunshu
   1. 你可能还需要停掉 yunshu 的其他组件，比如：
   ps aux | grep yunshu
    2. 如果还有 /opt/apps/yunshu/files/bin/xxx 等相关进程：
    pkill -f yunshu
    3. 如果你不再需要 YunShu，可以
    sudo rm -rf /opt/apps/yunshu

# 1. nostr的工作方式
1. 由两个部分组成：客户端和中继器。每个用户运行一个客户端。每个人都可以运行一个中继器。
2. 用户的身份用public key标识。每个帖子都有签名。每个客户端都会验证这个签名。
3. 每个客户端根据自己的选择从中继器接收数据，并且向其他选择的中继器发送数据。中继器不会与其他中继器对话，只会直接对用户。
4. 比如，为了关注某人，用户只需指示他们的客户端查询它知道的中继器以获取来自这个public key的帖子。
5. 在启动时，客户端从它知道的所有中继器中查询它所关注的所有用户的数据（例如从前一天的所有更新），然后按时间顺序向用户显示该数据。
6. 帖子可以包含任何类型的结构化数据，但最常用的数据将形成标准，以便所有客户端和中继器可以无缝地处理它们。

# 2. nostr如何解决其他网络无法解决的问题
## 2.1 **用户被禁和服务器被关闭**
- 中继器可以阻止用户在它那里发布任何内容，但这对他们没有影响，因为他们仍然可以发布到其他中继器。由于用户是通过公钥识别的，因此当他们被禁止时，他们不会失去他们的身份和他们的追随者基础。
- 不需要用户手动输入新的中继器地址（虽然这也应该被支持），每当你关注的人发布服务器推荐时，客户端应该自动将其添加到它将查询的中继列表中。
- 如果有人正在使用中继发布他们的数据但是想迁移到另一个中继，他们可以发布一个服务器推荐到之前的中继然后去。
- 如果某人被很多中继禁止，以至于他们无法广播他们的服务器推荐，他们仍然可以通过其他方式让一些亲密的朋友知道他现在正在用哪个中继发布。然后，这些亲密朋友可以向该新服务器发布服务器推荐，慢慢地，被禁用户的老粉丝群将开始从新中继继续找到他们的帖子。
	- 以上所有内容对于中继停止运行时也有效。
## 2.2 **抗审查**
- 每个用户都可以将他们的更新发布到任意数量的中继
- 中继可以向用户收取费用（该费用的协商目前不在协议范围内）以在此处发布，以确保抗审查性（总会有一些俄罗斯服务器愿意拿你的钱来换取服务你的帖子）
## 2.3 **垃圾邮件**
- 如果垃圾邮件是中继的一个问题，它可以要求支付发布费用或者其他形式的身份验证，例如电子邮件或电话，并将这些在内部与公钥关联，然后发布到该中继——或者使用其他的垃圾邮件屏蔽技术，比如哈希现金或者验证码。如果中继被用作垃圾邮件向量，它很容易被客户端取消列出，客户端可以继续从其他中继获取更新。
## 2.4 **数据存储**
- 为了让网络保持健康，不需要数百个活跃的中继。事实上，考虑到可以创建新的中继并在网络中轻松传播以防止有中继开始出现异常的事实，它只需要少数几个就可以正常工作。因此，所需的数据存储量总体上相对少于Mastodon或类似软件。
- 或者考虑一个不同的结果：其中存在数百个由业余爱好者运行的利基中继（高度专门化的中继），每个用来转发来自一个小群体的更新。该体系结构也可以扩展：数据从用户发送到单个服务器，然后从该服务器直接发送到使用该数据的用户。它不必由任何其他人存储。在这种情况下，任何一台服务器处理来自其他服务器的更新都不是很大的负担，拥有业余服务器也不是问题。
## 2.5 视频和其他大文件
- 中继很容易拒绝大型文件，或者对接收和托管大型内容收费。当信息和激励措施明确时，市场力量很容易解决问题。
## 2.6 欺骗用户的技巧
- 每个客户端都可以决定如何更好地向用户显示帖子，因此始终可以选择以你想要的方式使用你想要的内容——从使用AI来决定你将看到的更新顺序到只按照时间顺序查看。

# 3. 常见问题FAQ
## 3.1 这很简单，但是为什么以前没人做？
不知道，但是我认为与以下事实有关：创建社交网络的人要么想要赚钱的公司，要么是想要完全没有服务器的P2P活动家。他们都没有看到Nostr使用的两个世界的特定组合。
## 3.2 我如何找到要关注的人？
首先，你必须了解他们并以某种方式获得他们的公钥，要么通过询问，要么通过在某处看到它的引用。进入Nostr社交网络后，你将能够看到他们与其他人的互动，然后你也可以开始关注这些人并与之互动。
## 3.3 我如何找到中继？如果我没有连接到其他人连接的相同中继会怎样？
你将无法与那个人交流。但总有一些蛛丝马迹的提示使得你的客户端软件（或你手动）知道如何连接到其他人的中继并与他们交互。未来还有其他关于如何解决这个问题的想法，但是我们永远不能保证完美的可达性，任何协议都不能。
## 3.4 我能否知道有多少人在关注我吗？
不能，但是如果中继以协议外的方式合作，你可以得到一些估计。
## 3.5 用什么激励人们运行中继？
这个问题具有误导性。 它假设中继是存在的免费哑管道，以便人们可以通过它们四处移动数据。 在这种情况下，是的，激励措施将不存在。 事实上，所有其他 p2p 网络堆栈中的 DHT 节点都可以这样说：人们运行 DHT 节点的动机是什么？
## 3.6 Nostr 使您能够在服务器中继之间移动或使用多个中继，但如果这些中继仅在 AWS 或 Azure 上，有什么区别？
今天全球有成千上万的 VPS 提供商，不仅有 AWS 或 Azure。AWS or Azure are exactly the providers used by single centralized service providers that need a lot of scale, and even then not just these two. For smaller relay servers any VPS will do the job very well.

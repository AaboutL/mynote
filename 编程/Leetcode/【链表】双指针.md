# tricks
## 1. dummy虚拟头结点
当需要创造一条新链表时，虚拟头结点可以简化边界情况的处理。比如合并两个链表、拆分一个链表。
示例：
	1. 21题：合并两个有序链表
	2. 86题：分割链表
	3. 23题：合并K个升序链表
	4. 19题：删除链表的倒数第N个结点（如果倒数第N个结点正好是头结点，则使用dummy指针很容易处理）
	
## 2. 快慢双指针
快指针走的速度是慢指针的N倍。针对链表遍历的优化，不需要多次遍历从而得到想要的结果。
示例：
	1. 876题：链表的中间结点
	2. 141：环形链表（判断是否有环）
	3. 142：环形链表2（判断环的起点）
## 3. 前后双指针
链表倒数第K个结点
示例：
	1. 剑指offer22：链表中倒数第K个结点
	2. 19题：删除链表的倒数第N个结点（需要先找出倒数第N+1个结点）
	
## 4. 交替遍历双指针
示例：
	1. 160题：相交链表（判断两个链表是否相交）
## 5. 如果先走一步再判断不容易处理，那么就先判断再走一步

# C++基础——面向对象程序设计
## 1. OOP核心思想：数据抽象、继承、动态绑定
```
class Quote{
public:
	std::string isbn() const;
	virtual double net_price(std::size_t n) const;
};
clsss Bulk_quote: public Quote{
public:
	double net_price(std::size_t) const override;
};

```
**数据抽象**：将类的接口与实现分离
**继承**：可以定义相似的类型并对其相似关系建模
**动态绑定**：在一定程度上忽略相似类型的区别，而以统一的方式使用它们的对象
### 继承
1. 派生类必须在其内部对所有重新定义的虚函数进行声明，可以加上virtual关键字，也可以不加。
2. C++11新标准允许派生类显示地注明它将使用哪个成员函数改写基类的虚函数，方法是在该函数的形参列表之后增加一个override关键字

### 动态绑定
1. 通过动态绑定，可以使用同一段代码分别处理Quote和Bulk_quote的对象，如下
	```
	double print_total(ostream& os, const Quote& item, size_t n){
	double ret = item.net_price(n);
	os << "ISBN: " << item.isbn() << endl;
	return ret;
	}
	```
因为函数print_total的形参item是基类Quote的一个**引用**，所以既能使用基类Quote的对象调用net_price()，也能使用派生类Bulk_quote的对象调用它。实际执行哪个版本的net_price()函数，将由实际传入print_total的对象类型决定。函数的运行版本由实参决定，即在运行时选择函数的版本，所以**动态绑定又称为运行时绑定**
2. 在C++语言中，当使用基类的引用（或指针）调用一个虚函数时，将发生动态绑定。

### 派生类和基类
1. **基类通常应该定义一个虚析构函数**
当delete一个动态分配（使用new操作符）的对象的指针时，将执行析构函数。如果该指针指向继承体系中的某个类型时，则有可能出现**指针的静态类型与被删除对象的动态类型不符的情况**。
例如，如果删除一个Quote* 指针，该指针可能实际指向了一个Bulk_quote类型的对象，这时编译器就必须清楚它应该执行的是Bulk_quote的析构函数。通过在基类中将析构函数定义成虚函数以确保执行正确的析构函数版本。如果基类的析构函数不是虚函数，则delete一个指向派生类对象的基类指针将产生未定义的行为。
2. **静态类型和动态类型**
当使用存在继承关系的类型时，如果表达式是引用或者指针，则存在静态类型和动态类型不一致的情况，例如上述的print_total函数，**item的静态类型**是Quote&，它的动态类型依赖item绑定的实参，动态类型直到在运行时调用该函数时才会知道。如果传递一个Bulk_quote对象给print_total，则item的动态类型是Bulk_quote。
3. **从派生类向基类的类型转换**
	可以把派生类的对象当成基类对象使用，也可以将基类的指针或者引用绑定到派生类对象中的基类部分上。
	```
	Quote item; //基类对象
	Bulk_quote bulk; //派生类对象
	Quote *p = &item; //p指向Quote对象
	p = &bulk; //p指向bulk的Quote部分
	Quote &r = bulk; //r绑定到bulk的Quote部分
	```
	这种转换称为派生类到基类的类型转换。编译器会隐式地执行派生类到基类的转换。
	这种隐式的特性意味着可以把派生类对象或派生类对象的引用 用到需要基类引用的地方；可以把派生类对象的指针用到需要基类指针的地方。当使用基类的引用（或指针）时，实际上并不清楚该引用（或指针）所绑定的对象的真实类型，可能是基类对象，也可能是派生类对象。
4. **不存在从基类向派生类的隐式类型转换**
	之所以存在从派生类到基类的隐式类型转换，是因为派生类对象都包含一个基类部分，基类的引用（或指针）可以绑定到该基类部分上。一个基类的对象可以以独立形式存在，也可以作为派生类对象的一部分存在。
	由于基类的对象可能是派生类的一部分，也可能不是，所以不存在从基类向派生类的自动类型转换。即使基类的引用（或指针）绑定到派生类对象上，也不能执行从基类到派生类的转换，如下
	```
	Bulk_quote bulk;
	Quote *itemP = &bulk;  //正确，动态类型是Bulk_quote
	Bulk_quote *bulkP = itemP; //错误，不能将基类类型转换成派生类类型
	```
5. 派生类需要（但不是必须）对虚函数重新定义，覆盖掉从基类继承的旧定义。当没有覆盖时，该虚函数的行为类似普通函数，派生类会继承其在基类中的版本
6. 当基类希望派生类覆盖（override）某些成员函数时，通常把这些成员函数定义为虚函数。当使用指针或者引用调用虚函数时，该调用将被动态绑定。
7. **继承与静态成员**：
	如果基类定义了静态成员，则在整个继承体系中，只存在该成员的唯一定义，不论从基类派生出多少子类，对每个静态成员来说，都只存在唯一的实例。
8. **防止继承发生**
	如果不希望被继承，则在类名后面跟一个final关键字。
9. **对象之间不存在类型转换**
	派生类向基类的自动（隐式）类型转换只对指针或者引用类型有效。在派生类类型和基类类型之间不存在这样的转换。
	
### 虚函数
1. 对虚函数的调用可能在运行时才被解析。动态绑定只有当通过指针或引用调用虚函数时才会发生。
	当通过一个普通类型（非引用和指针）的调用虚函数时，不是动态绑定，在编译时就能确定调用的版本。
2. 一个派生类如果覆盖了继承而来的虚函数，则它的型材必须与被它覆盖的基类函数完全一致。
	派生类中虚函数的返回值类型一般也必须与基类函数匹配。但是有一个例外：**当类的虚函数返回类型是类本身的指针或引用时，此规则无效**
3. 虚函数与默认实参
	虚函数可以有默认实参，发生函数调用时，默认实参的值由本次调用的静态类型决定。即，如果通过基类的引用或指针调用函数，则使用基类中定义的默认实参，即使实际运行的是派生类中的函数版本。如果虚函数使用默认实参，则基类和派生类中定义的默认实参最好一致。
4. 回避虚函数的机制
	当对虚函数的调用不希望进行动态绑定，而是强迫其执行虚函数的某个特定版本，可以通过作用域运算符实现：
	```
	double undiscounted = baseP->Quote::net_price(42); //强行调用Quote中的net_price函数，而不管baseP实际指向的对象类型是什么。该调用在编译时完成
	```
	使用场景：当一个派生类的虚函数调用它覆盖的基类的虚函数版本时。此情况下，基类的版本通常完成继承层次中所有类型都要做的共同任务，而派生类中定义的版本需要执行一个与派生类本身密切相关的操作。
	
### 抽象基类
1. 纯虚函数与抽象基类
	1. 可以为纯虚函数提供定义，只是函数体必须定义在类的外部。
	2. 含有纯虚函数的类是抽象基类。抽象基类负责定义接口，后续的其他类可以且必须覆盖该接口。
	3. 不能直接创建一个抽象基类的对象。
2. 派生类构造函数只初始化它的直接基类。

# 类的静态成员变量
在不适用全局变量的情况下，使用静态成员变量可以实现同一个类的多个对象之间的数据共享。
1. 静态成员变量必须初始化，而且只能在类体外初始化。
2. 静态成员变量属于类，不属于某个类的对象。
	1. 无论创建多少个对象，静态成员变量只分配一块内存，当某个对象修改了静态成员变量，也会影响其他对象。
	2. 静态成员变量不占用对象内存，而是在对象外开辟内存。不创建对象也可以访问。
3. 编译时，在静态数据区分配内存，到程序结束时才释放。
4. 静态成员变量初始化不赋值，会被默认初始化，一般为0。静态数据区的变量都有默认初始值，而动态数据区（堆区、栈区）的变量默认是垃圾值。
5. 静态成员变量可以被所在类的派生类的对象共享。
6. 静态成员变量可以作为成员函数的默认参数，而普通成员变量不行。
	```
	class base{ 
	public: 
		static int staticVar; 
		int _var; 
		void foo1(int i = staticVar);   //正确,staticVar为静态数据成员 
		void foo2(int i = _var);    //错误,_var为普通数据成员 
	};
	```
	
7. 静态成员变量的类型可以是所属类的类型，但是普通成员变量不行。普通成员变量只能声明为所属类类型的指针或引用。
	```
	class base{ 
	public: 
		static base _1object;//正确，静态数据成员 
		base _2object;//错误 
		base *pObject;//正确，指针 
		base &mObject;//正确，引用 
	};
	```
	
# 静态成员函数
1. 静态成员函数只能访问类的静态成员变量。普通成员函数可以访问所有成员变量
2. 静态成员函数没有this指针。调用对象的非静态成员函数时，系统会把当前对象的起始地址赋给this指针，而静态成员函数不属于某一对象，所以无this指针。
3. 如果类的成员函数想作为回调函数来使用，一般情况下只能将其定义为静态成员函数。

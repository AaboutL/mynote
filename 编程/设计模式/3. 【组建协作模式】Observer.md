## 组件协作模式

组建协作模式通过晚绑定来实现框架与应用程序之间的松耦合，是这两者之间协作的常用模式。

典型的组建协作模式包括

1.  Template Method（模板方法）
2.  Strategy（策略模式）
3.  Observer/Event

## observer模式
### 动机
1. 在软件构建过程中，需要为某些对象建立一种“通知依赖关系”——一个对象（目标对象）的状态发生改变，所有的依赖对象（观察者对象）都将得到通知。如果这样的依赖关系过于紧密，将使软件不能很好地抵御变化。
2. 使用面向对象技术，可以将这种依赖关系弱化，并形成一种稳定的依赖关系，从而实现软件体系结构的松耦合。

### 模式定义
定义对象之间的一种一对多（变化）的依赖关系，以便当一个对象（subject）的状态发生改变时，所有依赖于它的对象都将得到通知。

#### 示例1 ：文件分割器
~~~
class MainForm: public Form{
		TextBox* txtFilePath;
		TextBox* txtFileNum; // 希望分割的文件个数
		ProgressBar* progressBar; //进度条对象，被通知的对象
public：
		void Button1_Click(){
				string filePath = textFilepath->getText();
				int number = atoi(textfileNum->getText().c_str());
				FileSplitter splitter(filePath, number, progressBar);
				splitter.split();
		}
}

class FileSplitter{
		string filePath;
		int fileNumber;
		ProgressBar* progressBar; //进度条，这里可能有各种各样的进度条：比如在界面程序中，是一个进度条；在终端中，是一个个的点。这里就是变化点。progressBar实际是一个通知，用一种抽象的方式表示通知，而不是用控件来表示。
public:
		FileSplitter(const string& filePath, int fileNumber, ProgressBar* progressBar):filePath(filePath), fileNumber(fileNumber),progressBar(progressBar){}
		
		void split(){
				//1. 读文件
				//2. 分批向小文件中写入
				for(int i = 0; i < fileNumber; i++){
						float progressVal = float(i+1)/float(fileNumber);
						progressBar->setValue(progressVal);//注意这个代码所在的类
				}
		}
}

//修改方式
//抽象出抽象的通知机制
class IProgress{
public:
		virtual void DoProgess(float value)=0;
		~IProgess(){}
}
class FileSplitter{
		string filePath;
		int fileNumber;
		IProgress* iProgress; //
public:
		FileSplitter(const string& filePath, int fileNumber, IProgress* iProgress):filePath(filePath), fileNumber(fileNumber),iProgress(iProgress){}
		
		void split(){
				//1. 读文件
				//2. 分批向小文件中写入
				for(int i = 0; i < fileNumber; i++){
						float progressVal = float(i+1)/float(fileNumber);
						iProgress->DoProgress(progressVal);
				}
		}
}

class MainForm: public Form, public IProgress{
		TextBox* txtFilePath;
		TextBox* txtFileNum; // 希望分割的文件个数
		ProgressBar* progressBar; //进度条对象，被通知的对象
public：
		void Button1_Click(){
				string filePath = textFilepath->getText();
				int number = atoi(textfileNum->getText().c_str());
				FileSplitter splitter(filePath, number, this);
				splitter.split();
		}
		
		virtual void DoProgress(float value){// 实现IProgress的虚函数
				progressBar->setValue(value);//注意这个代码所在的类，从FileSplitter转移到了MainForm中。使得FileSplitter不再依赖任何界面相关程序，而MainForm和ProgressBar本来就是相关的界面和控件。
		}
}

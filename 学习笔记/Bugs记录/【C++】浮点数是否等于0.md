浮点数类型double和float中的0都是近似0，所以不能直接与0比较是否相等。如
```
float a = 2.1;
if(a == 0) ...

double b = 1.2;
if(b == 0) ...
```
如果必须与0进行比较，则应该转化为判断浮点数是否在0附近的一个极小区间内，如
```
#define EPSILON 0.000000000001;
double a = 1.2;
if(a < EPSILON && a > -EPSILON) ...
// 即判断a是否在（EPSILON，-EPSILON）这个区间中。
```

# Bug说明
在构建g2o图优化时，判断一个3D点是否有效，是否应该加入到图中，用到的判断就是这个3D点的z坐标是否等于0。
在前面的代码中，特征点构造过程中会把其3D位置默认设置为（0, 0, 0）.
所以，这样就导致优化之前计算的重投影误差有时候正常，有时候却很大。一开始以为是特征跟踪，导致有些跟踪错了。后来把每个点的重投影无法打印出来，发现本来已经被重投影误差过滤掉的点，又出现在了g2o优化中，这样才查出了浮点数和0比较的问题。
## 建议
尽量避免让浮点数和0比较是否相等，应该通过其他的方法处理。
在这个bug中，本来已经在前面的代码中设置了一个3D点是否存在的标志了，那么最好就用这个标志的判断来决定这个点是否加入图中。
